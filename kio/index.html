From silicon to store.

Layer 01 · Edge compute
Live

FKAP1 + FKAP2 SoC.

Cortex-A55 / A76 cores · NPU 1-6 TOPS · Android 13/14 + Linux. On-device barcode + RFID processing.

Layer 02 · Display
Live

21.5″ FHD touch.

Capacitive touch · 1920×1080 IPS · 500 nits · 178°/178° viewing · 20° tilt adjustable.

Layer 03 · Peripherals
Live

Modular per-store.

2D CMOS barcode (EAN/UPC/Code128/PDF417/QR) · 80 mm thermal printer · status LED · camera · POS mount · RFID hard-tag remover + pin collector + 40-100L bin.

Layer 04 · Software
Live

StoreOS + Swift + FSM.

StoreOS sync (real inventory + pricing) · Fynd Swift Module for OTA firmware + remote diagnostics · FSM (Field Service Management) for service cycles.

Threads in active execution as of 01 - May - 2026. Each carries a Building pill: work is well-defined, owner is named, gating step is in motion.

Fashion Factory · 168-unit rollout
Building

Staged install across FY26-27.

168 production units locked for Reliance Retail Fashion Factory format. Hardware build + Swift Module preconfiguration in parallel; staged store-by-store install across FY26-27.

RFID-bin checkout flow
Building

Bulk drop-off for fashion stores.

40-100L volumetric RFID bin · paired with EAS hard-tag remover and pin collector. Customer drops a basket of tagged garments; bin reads the lot, kiosk surfaces totals for pay-and-go. Anchor format: Fashion & Apparel.

Virtual Try-On camera module
Building

Camera slot for VTO + on-kiosk preview.

Camera module slotting into Fashion & Apparel deployments. Pairs with Fynd Horizon for fit preview at the same surface where the customer pays. Adds Horizon as the upstream try-on surface.

FSM service-cycle automation
Building

Diagnostic alerts → proactive field service.

Swift Module emits diagnostics; FSM (Field Service Management) routes service tickets before downtime. Validated in SnacAdda; hardening for Fashion Factory scale. Diagnostic uptime is a tracked metric.

VI.SNACADDA · INTERNAL LAB CASE STUDY · Self-Checkout006 / 007
