01 · The job to be done
The pain
Brand video for social and storefront is the most expensive and slowest creative artefact a retailer ships. A single 15-30 s product reel runs through brief → storyboard → studio → shoot → edit → grade → CTA design — typically 3-6 weeks per format per brand, with talent and post costs that scale linearly with volume. Reliance's brand portfolio needs this output weekly across 100s of SKUs; the manual pipeline cannot.
What the agent does
Produces shoppable, on-brand video end-to-end. Trend Scout watches formats in market; Ralph Ideation drafts a brief; Brand Analyzer keeps style discipline; Script Writer turns the brief into a shot-by-shot storyboard; the frame stage generates first/last-frame stills using Vertex AI Imagen (AI Photoshoot output planned as scene reference); Veo or Kling animates each scene; FFmpeg post-production stitches text overlays and an AI-generated CTA frame; the Shoppable widget feeds the output into storefronts with product hotspots and analytics. Each stage is a durable workflow with retries and resumable state.
What humans still do
Brand reviews the storyboard at the Storyboard Review gate; can edit or reorder scenes inline. Performance Agent flags weak signals after publish; the human marketer decides which learnings feed back into the brief library. Trend selection and brand-safety overrides stay human.
02 · How it works · the closed loop
Inputs (blue, left · trend signals · brand DNA · product catalogue with planned AI Photoshoot integration · brief library · performance feedback) → 6 durable stages (centre · grey, with the orange Storyboard Review gate as the human-in-loop checkpoint) → Outputs (green, right · storyboard data · per-scene clips · shoppable feed · performance traces). Performance loop closes back to the brief library.
01
Brief intake
Brand approves a brief — sourced from chat ideation, a tracked trend, or a template. The brief kicks off the content production workflow.
02
Script + storyboard
Script Writer agent runs in multi-scene mode — generates narrative arc, scenes, visual treatment and captions. Routes through Claude (primary) with OpenAI / Gemini as backup.
03
Frame generation
Per-scene first/last-frame stills generated via Vertex AI Imagen or fal.ai. Reference images sourced from the product catalogue today; AI Photoshoot integration on the roadmap. The grader scores quality checks and auto-retries on fail.
04
Video animation
Per-scene Veo (Vertex · 4-8 s per clip) or Kling (fal.ai · 5-10 s per clip) animation between first and last frame · 9:16 or 16:9 · 720p or 1080p. Long-form videos fan out parallel scene generation; assembled with cross-fade scene transitions.
05
Post-production
Video editor overlays text, colour-grades, applies film grain, and stitches an AI-generated CTA frame — single product card, multi-product grid, or brand outro. Non-blocking — original video is used if post-production fails.
06
Distribution
Output ships to cloud storage, then served through the Shoppable feed (cached for 60-second freshness) with product hotspots and analytics tracking. Per-brand API keys exchanged for short-lived (15-minute) access tokens.

04 · Design patterns used
Multi-Agent Orchestration
6 specialist sub-agents (Script Writer · Trend Scout · Brand Analyzer · Ralph Ideation · Performance · Brief Suggestion) coordinated as durable workflows.
Tool Calling
Ralph Ideation Agent uses tool-use loops (search products · update draft · select product) to draft briefs against the live catalogue.
Wide Research
Long-form videos fan out parallel frame and video generation per scene (max 3 concurrent), then converge in video assembly.
Context Engineering
The reference processor categorises reference images (product · character · style) and prepares multimodal inputs for Gemini · brand-style context injected per script run.
Agent Memory
Brief library + brand DNA + performance traces persist as semantic + episodic memory · Performance Agent learnings feed back into the next brief.
Observability
Langfuse traces every AI, image and video call · grading reports trace back through the full pipeline · root-cause analysis runs on failures.
Evals
Per-scene frame grading (quality checks · auto-retry on fail) · prompt eval test suite · 68% backend test-coverage gate enforced in CI.
05 · Evals · what gates each output

Three layers run today on the Fynd-product side. Layer 1 · per-asset frame grading — the grader scores every generated frame against quality checks (colour accuracy · print fidelity · silhouette · category compliance); auto-retries on fail; ships only when checks pass. Layer 2 · trace lineage — Langfuse captures every AI, image and video call; the grader reconstructs full pipeline lineage from any grading report. Layer 3 · post-publish performance — Performance Agent reads engagement, CTR and attributed sales; learnings feed back into the brief library so the next brief starts smarter.

Gates and thresholds.

  • Frame grade passes per-scene quality checks before the scene goes to video
  • Storyboard Review gate · brand approves the storyboard before any video call (cost gate)
  • Post-production failure is non-blocking — original video ships if overlays fail
  • AI provider resilience · automatic fallback chain (Claude → OpenAI → Gemini → Vertex) when any provider fails
  • 68% backend test-coverage gate enforced in CI · separate eval suites for prompt regression
