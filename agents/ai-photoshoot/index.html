01 · The job to be done
The pain
Catalogue photoshoots gate every retail launch. Studio booking, model availability, stylists and post-production stretch each category drop by weeks. Lingerie alone runs 3-4× standard cost on talent and lighting, with turnarounds of 4-6 weeks. Volume scales linearly with cost.
What the agent does
Generates on-model catalogue images from flat-lay or mannequin reference shots. Three modes — Instant (2-4 minutes per set), Batch (cost-optimised bulk), and Single Image (on-demand QC fixes). Agentic across accessory placement, lighting extraction, autonomous issue detection and surgical Auto-Edit. Output lands on GCS with Ajio-standard naming, ready to list.
What humans still do
Brand QC reviews any image scoring below 7.0 (routed to Manual QC). Category onboarding remains human-led for new pods. Final approval before go-live to Ajio stays with the brand cataloguing team.
02 · How it works · the closed loop
Inputs (blue, left) → Workflow 10 steps (centre, grey) with explicit Detect → Edit → Check loop (orange, max 2 retries) → Outputs (green, right). Below-7.0 score routes to Manual QC (purple).
01
Upload
Product reference images (front · back · side) uploaded to the platform. Supports clothing, footwear and lingerie.
02
Analyse
Product image views classified (front · back · side · three-quarter). Garment details extracted: fabric · colour · pattern · silhouette.
03
Avatar & Pose
User selects avatars (gender · age · ethnicity) and poses from the pool. 12 camera view types available — front · back · side · full-angle · closeup · angled.
04
Background & Lighting
Background image selected. Lighting characteristics (direction · warmth · type) extracted to blend the model into the scene.
05
Prompt
Natural language prompt assembled per pose: product + avatar + theme + view-aware lighting + filtered accessories. Polished for flow and grammar.
06
Generate
On-model images generated in parallel batches (up to 20 concurrent). Background reference image passed alongside the prompt. Supports 4K output.
07
Detection
Unified Detection Agent scans each generated image against product references. Detects colour · logo · print · fabric · pose mismatches with code execution for zoom/crop inspection.
08
Auto-Edit
Edit Agent fixes detected issues with bounding-box-guided surgical editing. Validate-retry loop with max 2 attempts per image. Identity and background preservation enforced.
09
Quality
Images scored — 10.0 base, deductions per issue. Score ≥ 7.0 → Approved. Below routes to Manual QC. Realism weighted: Human 55% · Product 30% · Scene 15%.
10
Deliver
Final images uploaded to GCS, resolution-optimised, available via CDN. Ajio-standard naming convention: style_id/color_id/product_id/MODEL.jpg.

05 · Evals · what gates each output

Every generated image is scored before delivery. Base 10.0; deduct = `total_issues × (avg_confidence / 100) × 1.5`. Score ≥ 7.0 → Approved and delivered. Below 7.0 → routed to Manual QC for brand review. Current crop-extraction pipeline runs at 8.06/10 average.

Gates and thresholds.

  • Auto QC ≥ 7.0 to ship
  • Identity preservation enforced — face change penalty −4.0 · background change −3.0 · new regression −2.0 each
  • Realism weighted: Human Realism 55% · Product Details 30% · Scene Composition 15%
  • Bands: 9.0-10.0 Exceptional · 8.0-8.9 Very good · 7.0-7.9 Good · below 7.0 needs correction
